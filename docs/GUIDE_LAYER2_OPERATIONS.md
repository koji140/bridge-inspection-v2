## 2. 写真登録の手順（Step by Step）

現場で撮影した写真をシステムに登録する手順です。写真1〜10枚程度をまとめて登録できます。

### Step 1：Webシステムを開く

1. Google ChromeでWebシステムのURLを開きます。
2. 画面上部に「🏞️ 写真登録」「🗺️ マップ」「📄 報告書」の3つのタブが表示されていることを確認します。
3. 「🏞️ 写真登録」タブが選択された状態（青くなっている）であることを確認します。

### Step 2：写真をアップロードする

1. 画面中央の点線で囲まれたエリア（「写真をここにドロップ または クリックして選択」と書かれた場所）に写真ファイルをドラッグ＆ドロップします。
2. または、そのエリアをクリックするとファイル選択ダイアログが開くので、写真を選択します。複数ファイルを同時に選択できます。
3. アップロードが始まると、写真ごとに「登録中...」という表示が出ます。
4. 全写真の処理が完了すると「✅ 登録完了」という表示に変わります。このとき右下に写真の一覧が表示されます。

※ JPG・PNG形式に対応しています。写真にGPS情報（EXIF）が含まれていると、地図上に自動でピンが立ちます。  
※ 1回のアップロードで10枚を超える場合は、数回に分けてください。

### Step 3：JSONをコピーしてChatGPTに渡す

1. 写真一覧の右上にある「📋 JSONをコピー」ボタンをクリックします。
2. 「📋 コピーしました」と表示されたら、クリップボードにJSONがコピーされています。
3. ChatGPTのカスタムGPT（「高架下点検マップ＆報告書システム」）を開きます。
4. 画面下部のテキスト入力欄に、コピーしたJSONをCtrl+Vで貼り付けます。
5. さらに同じ写真ファイルをChatGPTにドラッグ＆ドロップしてアップロードします。
6. メッセージを送信します（Enterキーまたは送信ボタン）。

※ JSONと写真の両方を渡すことで、ChatGPTがどの写真がどのIDかを正確に把握できます。これを省略するとIDと写真の対応がずれる可能性があります。

### Step 4：AIの提案を確認・修正して保存する

1. ChatGPTが各写真について「所見」「タグ」「対応要否（1〜5）」を提案します。
2. 内容を確認して、必要であれば修正をChatGPTに指示します（例：「No.2の所見を〇〇に変更してください」）。
3. 内容に問題がなければ「保存してください」と入力します。
4. ChatGPTが保存処理を実行します。「保存しました」と表示されたら完了です。
5. Webシステムのマップタブに戻り、新しいピンが地図上に表示されていることを確認します。

※ 保存後にマップに表示されない場合は、Ctrl+Shift+Rでブラウザをリロードしてください。

⚠️ 「保存してください」と明示的に指示するまで、ChatGPTはデータを保存しません。確認を省略しないでください。

## 3. 地図の見方・ピンの確認

### Webシステムの「🗺️ マップ」タブを開く

地図には橋脚ごとにピンが立っています。ピンの色で対応状況が一目でわかります。

| ピンの色 | 意味 |
|---|---|
| 🔴 赤 | 要対応（初期対応未済）の写真が含まれる。最優先で確認が必要 |
| 🟠 オレンジ | 要対応（初期対応済・以降要）の写真が含まれる |
| 🟡 黄 | 要対応（初期対応済・以降不要）の写真が含まれる |
| 🔵 青 | 現地確認の結果、異常なし |
| ⚫ 灰 | 対応不要事象のみ |
| ○ 薄灰 | 対応要否が未設定 |

**ピンの右上に青いバッジ（数字）が表示されている場合**

「未報告あり」を意味します。報告書にまだ掲載されていない写真が存在することを示しています。

### ピンをクリックして写真を確認する

1. 地図上のピンをクリックすると、画面右側に写真一覧が表示されます。
2. 各写真カードに「photo_id」「対応要否」「状態」「撮影日時」「橋脚番号」「タグ」「所見」が表示されます。
3. 写真をクリックすると写真が拡大表示されます（ESCキーまたは画面外クリックで閉じます）。
4. 「編集」ボタンを押すと橋脚番号・対応要否・タグ・所見を修正できます。
5. 「削除」ボタンを押すと論理削除されます（報告書確定済みの写真は削除できません）。

## 4. 報告書の作成手順

### Step 1：報告書タブを開く

1. Webシステムの「📄 報告書」タブをクリックします。
2. 左側に報告書の一覧が表示されます。最初は「報告書はまだありません。」と表示されます。

### Step 2：新規報告書を作成する

1. 左上の「＋ 新規作成」ボタンをクリックします。
2. 画面右側にフォームが表示されます。
3. 「対象ピン（複数選択可）」の一覧から、今回の報告書に含める橋脚のチェックボックスをオンにします。
4. 500ミリ秒ほど待つと「実施日」「実施場所」「内容」が自動で入力されます（AIによる提案）。
5. 自動入力された内容を確認し、必要に応じて手動で修正します。
6. 「報告書種別」を選択します（定期点検 / 臨時点検 / 災害後確認 / 補修後確認 / その他）。
7. 「作成」ボタンをクリックします。

※ 「実施日」は選択した橋脚の写真の中で最も新しい撮影日が自動入力されます。  
※ 「実施場所」と「内容」はOpenAI APIが橋脚番号と所見をもとに自動提案します。内容が合わない場合は手動で修正してください。

### Step 3：PDFを生成する

1. 報告書が作成されると左側の一覧に表示されます。クリックして選択します。
2. 右側に報告書の詳細が表示されます。内容を確認します。
3. 「✏️ 内容を編集」ボタンから実施場所・内容等を修正できます（確定前のみ）。
4. 「📄 PDF生成」ボタンをクリックします。
5. PDF生成には10〜30秒ほどかかります。完了するとPDFのリンクが表示されます。
6. リンクをクリックしてPDFの内容を確認します。

### Step 4：報告書を確定する

1. PDFの内容に問題がなければ「✅ 報告書を確定する」ボタンをクリックします。
2. 確定すると報告書のステータスが「confirmed（確定済み）」になります。
3. 確定した報告書に含まれる写真のステータスが「published（掲載済み）」になります。

⚠️ 確定後は報告書の内容を変更できません。内容に問題がある場合は確定前に修正してください。  
⚠️ 確定後に写真を削除・編集することもできなくなります。

## 5. 「正常な状態」とはどういう状態か

異常を判断するために、正常な状態を把握しておく必要があります。

| 確認項目 | 正常な状態 |
|---|---|
| マップのピン表示 | 登録済みの橋脚がピンとして地図上に表示される。写真登録後にCtrl+Shift+Rでリロードすると新しいピンが出る |
| 写真登録の完了表示 | 「✅ 登録完了」と表示される。エラーメッセージが出ていない |
| JSONコピー後のChatGPT連携 | ChatGPTが各写真について所見・タグ・対応要否を提案する。「処理できません」等のエラーが出ていない |
| 報告書の自動提案 | ピン選択後5〜15秒で実施場所・内容が自動入力される（AIの処理時間のため少し待ちます） |
| PDF生成 | 「PDF生成」ボタン押下後10〜30秒でPDFリンクが表示される |
| エラー通知メール | 届いていない = 正常。届いた場合は次章の対応手順を参照 |

## 6. エラーが起きたときの対応手順

### 6-1. エラー通知メールが届いたとき

エラー通知メールには「発生箇所」と「エラー内容」が記載されています。以下の対応表を参照してください。

| エラー内容（メールに記載） | 原因 | 対処法 |
|---|---|---|
| OpenAI APIエラー: Incorrect API key | APIキーが無効または変更された | ⚙️設定シートのOPENAI_API_KEYを確認。石丸に連絡してAPIキーを再発行してもらう |
| OpenAI APIエラー: RESOURCE_EXHAUSTED | APIの利用上限に達した | platform.openai.comにログインしてUsageを確認。石丸に報告する |
| 「⚙️設定」シートが見つかりません | SpreadsheetのシートがRenameされた | Spreadsheetを開いてシート名が「⚙️設定」になっているか確認。なければ石丸に連絡 |
| LOCK_MISMATCH | 同じデータを複数人が同時に編集した | 画面をCtrl+Shift+Rでリロードして再操作。繰り返す場合は石丸に連絡 |
| 上記以外のエラー | 技術的な問題 | エラーメールの内容をそのまま石丸（k.ishimaru5532@shutoko-assoc.jp）に転送 |

### 6-2. よくある操作トラブル（FAQ）

**Q：写真を登録したのに地図にピンが表示されない**

以下の順で確認してください。

1. Ctrl+Shift+Rでブラウザをハードリロードする（最もよくある原因）
2. 写真にGPS情報が含まれているか確認する（GPS情報がない写真はピンが立たない場合がある）
3. ブラウザの開発者ツール（F12キー）を開いてConsoleタブにエラーが出ていないか確認する
4. 上記で解決しない場合は石丸に連絡する

**Q：ChatGPTがphoto_idと画像を混同しているようだ**

この状態でそのまま保存すると、間違った橋脚に間違った所見が登録されます。絶対に保存しないでください。

1. ChatGPTに「最初からやり直してください」と入力する
2. JSONを貼り付け直して、写真も再度アップロードしてやり直す
3. 繰り返し発生する場合は石丸に連絡する

⚠️ photo_idと画像の対応がずれたまま保存すると、点検記録の信頼性が損なわれます。必ず確認してから保存してください。

**Q：報告書のPDF生成が失敗する**

1. GASエディタを開いて「実行数」タブでエラーメッセージを確認する
2. 報告書に紐づく写真が1枚以上あるか確認する（写真がないとPDF生成できない）
3. 上記で解決しない場合は石丸に連絡する

**Q：報告書の自動提案（実施場所・内容）が出ない**

1. ピンを選択後、最大15秒ほど待つ（AIの処理に時間がかかる）
2. ⚙️設定シートのOPENAI_API_KEYが正しいか確認する
3. Spreadsheetの「📝プロンプト管理」シートにSUGGEST_REPORT_HEADERのプロンプトが入っているか確認する
4. 上記で解決しない場合は、手動で実施場所・内容を入力して報告書を作成する（自動提案なしでも作成可能）

## 8. プロンプト・設定値の変更方法

コードを修正せずにSpreadsheetのシートから変更できます。再デプロイも不要です。

### AIのプロンプトを変更したい

1. Spreadsheetを開いて「📝プロンプト管理」シートをクリックします。
2. SUGGEST_REPORT_HEADERの行のB列（プロンプト本文）を直接編集します。
3. セルの外をクリックして保存します。次回の処理から新しいプロンプトが使われます。

※ `{{橋脚番号}}` と `{{所見一覧}}` はコードが実際のデータに置き換えるプレースホルダーです。削除しないでください。

### OpenAIのモデルを変更したい

1. Spreadsheetの「⚙️設定」シートを開きます。
2. OPENAI_MODELの行のB列の値を変更します（例：gpt-4o-mini → gpt-4o）。
3. セルの外をクリックして保存します。次回の処理から新しいモデルが使われます。

### エラー通知先メールアドレスを変更したい

1. Spreadsheetの「⚙️設定」シートを開きます。
2. ERROR_NOTIFY_EMAILの行のB列の値を新しいメールアドレスに変更します。
3. セルの外をクリックして保存します。

## 9. コード修正が必要なときはClaudeに相談する

「エラーが直らない」「新しい機能を追加したい」「設定を変えてもうまくいかない」など、技術的な問題が起きたときはAI（Claude）に相談することで解決できます。石丸さんがこのシステムを作ったのも、Claudeと対話しながら開発したためです。

⚠️ Claudeを使うときは必ず「引き継ぎキット」を読み込ませてください。キットなしで質問すると、このシステムの文脈を知らない状態で答えるため、的外れな回答になります。

### Step 1：Claudeのアカウントを用意する

1. ブラウザで claude.ai を開きます。
2. Googleアカウントでサインインします（無料プランでも使えますが、Pro（月額$20）の方が回答の質が高く、ファイルをより多く読めます）。
3. ログイン後、画面左側に「新しいチャット」と「プロジェクト」のメニューが表示されます。

### Step 2：プロジェクトを開く（または作成する）

「プロジェクト」とは、複数のファイルをまとめて読み込んだ状態でClaudeと会話できる機能です。このシステムの引き継ぎキットがすでにプロジェクトに登録されています。

1. 画面左側の「プロジェクト」をクリックします。
2. 「高架下点検マップ＆報告書システム」というプロジェクトが表示されていればそれをクリックします。
3. プロジェクトが見つからない場合は、石丸（k.ishimaru5532@shutoko-assoc.jp）に連絡してプロジェクトへのアクセス権を付与してもらうか、後述の「プロジェクトを新規作成する手順」を参照してください。

### Step 3：Claudeに質問する

プロジェクトを開いた状態でチャットを始めると、ClaudeはSKILL.mdをはじめとする引き継ぎキット全体を読んだ状態で回答します。

質問の例：

- 「GASのコードを修正したい。PDF生成でエラーが出ている。エラーメッセージは〇〇です。どこを直せばいいか教えてください」
- 「報告書の自動提案が動かなくなった。原因と対処法を教えてください」
- 「OpenAI APIのモデルをgpt-4oに変えたい。何をすればいいか手順を教えてください」
- 「新しい機能として〇〇を追加したい。どう実装すればいいか教えてください」

Claudeはコードの修正案を出してくれます。「これをGASエディタに貼り付けて再デプロイしてください」という指示に従えば、コードが読めなくても対応できます。

※ Claudeはこのシステムの設計判断の背景・過去に発生したエラーのパターン・GASの落とし穴まで把握しています。困ったことは何でも聞いてください。

## 10. システムを変更したときのログ記録

プロンプトや設定を変更したときは、必ずSpreadsheetの「変更履歴」シートに記録してください。後から「いつ・何を・なぜ変えたか」を追跡できるようにするためです。

| カラム | 記入内容 |
|---|---|
| 変更日 | 変更した日付（自動入力されることもある） |
| バージョン | 例：v2.2（前のバージョンより1上げる） |
| 変更者 | 自分の名前 |
| 変更内容 | 何をどう変えたか（例：「SUGGEST_REPORT_HEADERのプロンプトに「です・ます調」の指定を追加」） |
| 影響範囲 | 変更が影響するもの（例：「報告書の自動提案」） |
| テスト確認 | 変更後に動作確認したか（✅ または ❌） |
