# Claude向けスライド生成プロンプト（共通テンプレート）

## 目的
GitHub上の正本ファイルを参照し、再利用可能な日本語プレゼンスライドを生成する。

## 実行前提
- GitHubは連携済み
- 正本ファイル（内容定義）を最優先で参照
- テンプレートはデザインのみ使用（配色・フォント・スタイル）
- スライド構成はこのプロンプトを優先

## 入力指定
- repository: （対象リポジトリURL）
- content正本: `docs/presentation/projects/<project-name>/slide-content.md`
- 補足資料: 必要に応じて `docs/presentation/<project-name>/slide-outline.md` 等

## 生成ルール（厳守）
- 原稿の文章をそのまま貼り付けない
- 1スライドは「見出し + 最大4 bullet」
- 1 bullet は1行で読める長さ
- 1スライド1メッセージ
- 長文・段落は使わない
- 図解・比較・フローを優先

## テンプレートの扱い
- 添付テンプレートはデザインのみ使用する
- 配色・フォント・スタイルのみ踏襲する
- スライド構成・ページ構成はこのプロンプトを優先する
- テンプレートの文章や章構成はコピーしない

## スライド型
- 通常スライド: 要点3〜4個
- 6枚目: As-Is / To-Be の業務フロー図（Mermaid）
- デモ3枚: スクリーンショット枠 + 要点3つ（仮画像禁止）

## 6枚目（最重要）
- 箇条書きではなく図で出力
- Mermaid形式で出力
- 上段: As-Is（横フロー）
- 下段: To-Be（横フロー）
- 各段4つの箱を左→右
- 上下で同じ構造

必ずこの形式で出力:

flowchart TD
  subgraph AsIs[As-Is]
    direction LR
    A1[写真] --> A2[整理] --> A3[報告] --> A4[共有]
  end
  subgraph ToBe[To-Be]
    direction LR
    B1[登録] --> B2[GPT] --> B3[地図] --> B4[出力]
  end

## デモスライド指定（3枚）
- タイトル
- 要点3つ
- 短い補足文
- 「スクリーンショットをここに配置」とわかる枠

## 出力形式
各スライドごとに以下で出力:

スライドタイトル:
- bullet
- bullet
- bullet
