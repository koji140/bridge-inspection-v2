## Layer 1：経営者・管理者向け

作成日：2026年3月　／　作成者：石丸 Koji（KAKEAI プロダクト本部長）

このドキュメントは、本システムの導入目的・効果・コスト・引き継ぎ体制を経営者・管理者が把握するために作成しました。

## 1. このシステムが解決する問題

首都高速道路の高架下点検業務では、現場で撮影した写真・所見・位置情報をそれぞれ別々に管理していたため、以下の問題が発生していました。

| 問題 | 具体的な影響 |
|---|---|
| 写真と所見が別ファイルに分散 | どの写真がどの橋脚のものか追跡しづらく、報告書作成時に突合に時間がかかる |
| 報告書作成が手作業 | 担当者が毎回フォーマットから作成するため、作業時間が長く、フォーマットのばらつきも生じる |
| GPS情報の活用不足 | 写真にGPS情報が埋め込まれているが活用できておらず、橋脚ごとの整理を手動で行っている |
| 対応要否の管理が属人化 | 各橋脚の対応状況（要対応・対応済み・対応不要等）を担当者の記憶や個別メモで管理している |

## 2. このシステムで何が変わるか

| 項目 | 導入前 | 導入後 |
|---|---|---|
| 写真管理 | フォルダに混在・手動整理 | GPS座標をもとに地図上でピン表示。橋脚ごとに写真が自動整理される |
| 所見・タグ入力 | 担当者が手入力 | AIが写真を見て所見・タグ・対応要否を自動提案。担当者が確認して保存 |
| 報告書作成 | 毎回手作業でフォーマット | 対象橋脚を選ぶだけで実施場所・内容をAIが自動提案。ボタン1つでPDF生成 |
| 対応状況の把握 | 担当者の記憶・個別メモ | 地図上のピン色で対応要否を一目で把握できる（赤=要対応、灰=対応不要） |
| 過去記録の参照 | ファイルを探す | 地図からピンをクリックするだけで過去の写真・所見を即座に確認できる |

## 3. システムの基本的な流れ（3ステップ）

現場担当者が行う操作は大きく3つに分かれます。

| ステップ | 誰が | 何をするか | 何が起きるか |
|---|---|---|---|
| ① 写真登録 | 現場担当者 | Webブラウザを開き、写真ファイルを画面にドラッグ＆ドロップする | 撮影日時・GPS座標が自動取得され、Google Driveに保存される。写真IDが発番されて管理台帳に登録される |
| ② 所見確認 | 現場担当者 | ChatGPTに写真をアップロードし、Webで生成されたIDリストを貼り付ける | AIが写真内容を解析して所見・タグ・対応要否を提案。担当者が内容を確認・修正して保存する |
| ③ 報告書作成 | 報告書担当者 | Webの報告書タブで対象橋脚を選択し「作成」→「PDF生成」を押す | AIが実施場所・内容を自動提案。確認後にPDF報告書が自動生成されGoogle Driveに保存される |

## 4. 運用コスト

| 項目 | 費用 | 備考 |
|---|---|---|
| Google Apps Script（GAS） | 無料 | Google Workspaceに含まれる |
| Google Drive（写真・PDF保存） | 無料〜 | 組織のDriveストレージ内に収まれば追加費用なし |
| OpenAI API（報告書の提案生成） | 従量課金 | 1回の報告書作成あたり約0.1〜0.3円程度（gpt-4o-mini使用時）。月数十件の処理であれば月数百円以下の見込み |
| ChatGPT Plus（所見提案） | 月額$20（約3,000円） | カスタムGPT機能を使用するために必要。1アカウントで運用可能 |
| Cloudflare Worker（API転送） | 無料枠内 | 月10万リクエストまで無料。現在の利用量では超過しない見込み |

※ 月間のメインコストはChatGPT Plus（約3,000円/月）とOpenAI API（数百円/月）の合計で、月4,000円以下が見込まれます。

## 7. 障害が起きたときの判断フロー

| 状況 | 最初の対応 |
|---|---|
| エラー通知メールが届いた | 運用担当者がLayer2「エラー対応表」を確認 → 解決しなければ石丸に連絡 |
| 写真が登録できない | 運用担当者がLayer2「FAQ」を確認 → 解決しなければ石丸に連絡 |
| 報告書が作れない | 同上 |
| OpenAI APIが使えない | 管理者がplatform.openai.comでAPIキーとクォータを確認 |
| コードの修正が必要 | 石丸（またはClaudeに引き継ぎキットを読ませて対応依頼） |
